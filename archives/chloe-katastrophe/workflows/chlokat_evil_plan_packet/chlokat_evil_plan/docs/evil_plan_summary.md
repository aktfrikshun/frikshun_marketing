# ChloKat / FrikShun Evil Plan Summary

## Brand Architecture

- FrikShun is the umbrella creative/engineering identity.
- Chloe Katastrophe is the virtual artist and fictional character.
- ChloKat is Chloe Katastrophe's nickname.
- Public ChloKat home: `chlokat.frikshun.com`.

The public site should feel like a FrikShun research archive where Chloe's memories, songs, images, videos, news, and transmissions are being reconstructed. It should also work as a superfan search engine: one place to find the latest Chloe news, public artifacts, releases, lore, links, and approved archive fragments.

The content creator operating system is a FrikShun tool, not the public ChloKat homepage. It can run locally first and may later become a web product for other creators managing fictional artists, virtual artists, social drafts, campaigns, and searchable public archives.

## Core Product Concept

Build two connected surfaces:

1. A public ChloKat archive/search/news site for fans.
2. A private FrikShun creator operating system for managing artifacts, drafts, campaigns, publishing workflows, and analytics.

Chloe is the first resident.

The system should:

1. Store artifacts.
2. Preserve canon.
3. Generate platform-native social drafts.
4. Publish or assist publishing to social platforms.
5. Create public archive pages.
6. Let fans search the archive.
7. Let fans talk to Chloe through approved canon.
8. Track which content creates engagement and superfans.

## Guiding Principle

Do not make Allen handcraft seven separate posts every time he creates content.

The system should turn one artifact into platform-appropriate drafts, each written for the native audience and purpose of that platform.
